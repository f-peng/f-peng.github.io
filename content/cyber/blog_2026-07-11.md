<h3 class="class_h3">Backdoor d'authentification cachée dans plusieurs firmwares Tenda (CVE-2026-11405)</h3>
- 📅 Sat, 11 Jul 2026 00:00:00 +0000
- 🔗 Source: https://cyberveille.ch/posts/2026-07-11-backdoor-d-authentification-cachee-dans-plusieurs-firmwares-tenda-cve-2026-11405/

<h2 id="-contexte">🔍 Contexte</h2>
<p>Le CERT/CC (Carnegie Mellon University) a publié le 6 juillet 2026 la note de vulnérabilité <strong>VU#213560</strong> concernant la découverte d&rsquo;une <strong>backdoor d&rsquo;authentification cachée</strong> dans plusieurs versions du firmware de routeurs et équipements réseau <strong>Tenda</strong>.</p>
<h2 id="-description-technique">🛠️ Description technique</h2>
<p>Le binaire du serveur web <code>/bin/httpd</code> contient une fonction <code>login()</code> qui, en cas d&rsquo;échec de l&rsquo;authentification normale (basée sur MD5), invoque <code>GetValue(&quot;sys.rzadmin.password&quot;)</code> pour récupérer un mot de passe alternatif depuis la configuration de l&rsquo;appareil. Une comparaison en <strong>clair via <code>strcmp()</code></strong> est ensuite effectuée entre le mot de passe fourni par l&rsquo;utilisateur et cette valeur. En cas de correspondance, un accès administrateur (<code>role=2</code>) est accordé avec création d&rsquo;une session valide.</p>

<h3 class="class_h3">GhostLock (CVE-2026-43499) : stack-UAF dans le noyau Linux depuis 15 ans, exploit stable à 97%</h3>
- 📅 Sat, 11 Jul 2026 00:00:00 +0000
- 🔗 Source: https://cyberveille.ch/posts/2026-07-11-ghostlock-cve-2026-43499-stack-uaf-dans-le-noyau-linux-depuis-15-ans-exploit-stable-a-97/

<h2 id="-contexte">🔍 Contexte</h2>
<p>Publié le 7 juillet 2026 par Nebula Security (nebusec.ai), cet article de recherche détaille <strong>GhostLock (CVE-2026-43499)</strong>, une vulnérabilité de type <strong>stack Use-After-Free (UAF)</strong> dans le noyau Linux, découverte par l&rsquo;équipe VEGA. La faille a été récompensée par Google à hauteur de <strong>92 337 USD</strong> dans le cadre du programme <strong>kernelCTF</strong>.</p>
<h2 id="-vulnérabilité">🐛 Vulnérabilité</h2>
<ul>
<li><strong>Type</strong> : Stack Use-After-Free (stack-UAF)</li>
<li><strong>Composant affecté</strong> : <code>kernel/locking/rtmutex.c</code>, fonction <code>remove_waiter()</code></li>
<li><strong>Versions affectées</strong> : Linux <strong>v2.6.39-rc1</strong> à <strong>v7.1-rc1</strong> (introduite en 2011 avec le rework rtmutex)</li>
<li><strong>Condition requise</strong> : <code>CONFIG_FUTEX_PI=y</code> uniquement — aucun privilège, aucun namespace utilisateur requis</li>
<li><strong>Cause racine</strong> : <code>remove_waiter()</code> efface <code>current-&gt;pi_blocked_on</code> au lieu de <code>waiter-&gt;task-&gt;pi_blocked_on</code> sur le chemin proxy (<code>rt_mutex_start_proxy_lock()</code>), laissant un pointeur dangling vers une stack frame libérée</li>
</ul>
